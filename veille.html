Ma veille Technologique

Pour ma veille Technologique j'ai choisir plusieurs sujets comme l'attaque NPM qui a eu lieu en Septembre 2025, les attaques “Dependency Confusion”, attaque sur GitHub Actions (GhostAction) et les attaques via Quishing.


phishing de Qix


En septembre 2025, Qix (Josh Junon) a eu son compte développeur npm compromis, il maintenait les librairies comme chalk, debug, ansi-style.


Cette attaque a eu lieu via du phishing. Et visait des portemonnaies crypto.


Les risques:

  • Vol de fonds crypto (via des redirections d’adresses)
  • Distraction difficile à détecter pour un utilisateur lambda, car le code malveillant agit lors de l’install ou au runtime du navigateur
  • Propagation via les dépendances : beaucoup de projets JavaScript / Node dépendent, directement ou indirectement, de ces paquets populaires
  • Confiance rompue dans la supply chain : un seul mainteneur compromis peut infecter un très grand nombre de projets

Les attaques Dependency Confusion


C’est une attaque où un pirate publie un package sur un registre public (npm, PyPI, etc.) ayant le même nom qu’un package interne (privé) utilisé par des entreprises. Orca Security

Les gestionnaires de paquets (npm, pip, etc.) cherchent d’abord sur les registres publics. Donc si un package public avec un nom identique existe, il peut être installé à la place du package privé légitime. Orca Security

Résultat : un code malveillant peut être injecté dans l’environnement de l’entreprise, car le package “trafiqué” vient d’un registre public, même si l’entreprise pensait utiliser un package interne.


Les risques:

  • Exécution de code malveillant dans des environnements d’entreprise
  • Exfiltration de données sensibles
  • Compromission de la chaîne de livraison logicielle (supply chain)
  • Difficulté de détection si les équipes ne surveillent pas les dépendances “externales” ou “privées” vs publiques

Les attaques github Actions


GhostAction est une campagne d’attaque découverte par GitGuardian (2025)

~327 comptes GitHub auraient été compromis, injectant worflows Github malveillant infectant ~817 dépots

Le worklow exfiltrait des secrets (jetons npm, PyPI, DockerHub, AWS, etc.) a un serveur controllé par les attaquants.


L’attaque a utilisé du code très simple (pas de dépendances externes) dans les workflows, ce qui la rendait difficile à repérer par des revues de sécurité rapides.

Le domaine d’exfiltration identifié : bold-dhawan.45-139-104-115.plesk.page selon GitGuardian.


L’attaque serait à l’origine d’un CVE (CVE-2025-30066) lié à l’Action “tj-actions/changed-files”.

Le point d’entrée aurait été un jeton d’accès personnel (PAT) compromis du bot @tj-actions-bot.


Le Quishing


Le Quishing est similaire a du Phishing, mais utilise des codes QR pour tromper les victimes.

Le qr code sera utilisé pour rediriger les gens vers un site qui se fait passer pour une entreprise légitime.

Cette page phishing peut demander des informations sensibles (identifiants, mot de passe, clé, paiement) ou déclencher un téléchargement.


Les risques:

  • Les utilisateurs sont souvent moins méfiants avec un QR code, car c’est “rapide” et “pratique”
  • Le phishing via QR est plus difficile à détecter pour certaines personnes (URL masquée)
  • Les attaques peuvent être combinées : QR → redirige vers un site qui installe un package malveillant
